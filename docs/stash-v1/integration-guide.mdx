## For crosschain DeFi

Sprinter Stash enables solvers to **borrow crosschain credit on-demand** to execute user intents without needing pre-funded inventory.

## Overview of the Stash Fill Lifecycle

```
Solver detects user intent
  → Solver previews estimated borrowing quote
    → Receive borrow quote estimate
      → Fill using Stash Credit?
          Yes → Solver reserves credit and authorizes the fill
            → Solver borrows liquidity from Sprinter Stash
              → Stash executes cross-chain swap/bridge
                → Intent protocol repays borrowed credit + costs
                  → Fill complete
          No  → Abort fill
```
